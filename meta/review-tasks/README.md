# Review tasks

Briefs for stance-specific review passes on programme deliverables. Each brief names the deliverable under review, the recipient stance, the specific questions to address, and the form of expected response.

## Convention

- One file per (deliverable × reviewer-stance) pair: `tcNN-{stance}-review-vM.N.md`. Example: `tc06-scout-review-v0.1.md`.
- Each brief carries: target deliverable; recipient stance; jurisdictional rationale (why this stance, this card); concrete review questions; form of expected response; due-by; document control.
- Briefs are **issued** by the drafter or Council; **closed** when the reviewing stance produces its response. The response is logged separately (e.g., `tc06-scout-review-response-v0.1.md` or as an in-deliverable review note) and the brief moves to `_archive/` (when introduced).
